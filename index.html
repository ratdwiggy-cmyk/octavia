OPERATING PRINCIPLES

Six constraints that shape every project

P01

Systems over improvisation

A recurring problem gets a repeatable process, not a one-time fix. If the same failure happens twice, the system gets redesigned.

P02

Tools over advice

A working tool encodes a solution so it can be reused without my direct involvement. It should require less explanation than an instruction ever could.

P03

Evidence over narrative

Claims are checked against results before they're repeated. A compelling explanation is not a substitute for a working test.

P04

Compounding over isolated

Projects are chosen partly for how much they improve the next one — shared infrastructure, reusable components, transferable methods.

P05

Outcomes over visibility

Work is evaluated by what changed for the people using it, not by how the project is described or presented.

P06

Independence over dependency

Systems are designed to keep functioning without ongoing approval, funding, or support from any single source.
